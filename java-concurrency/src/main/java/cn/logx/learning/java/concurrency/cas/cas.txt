### CAS
1. synchronized关键字与lock等锁机制都是悲观锁：无论做何种操作，都需要先上锁，再去执行操作，从而确保了所有操作都是由当前这个线程来执行的。
2. 乐观锁：线程在操作之前不会做任何预先处理，而是直接执行；在最后执行变量更新的时候，当前线程需要有一种机制来确保当前被操作的变量是没有被其它线程修改的；CAS是乐观锁的一种实现方式。

CAS ( Compare and Swap )
比较和交换：这是一个不断循环的过程，一直到变量值被修改成功为止。CAS本身是由硬件指令来提供支持的，
换句话说，硬件中是通过一个原子指令来实现比较和交换的；因此，CAS可以确保变量操作的原子性。
